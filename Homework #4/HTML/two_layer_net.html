def rel_error(x, y):
    """ returns relative error """
    return np.max(np.abs(x - y) / (np.maximum(1e-8, np.abs(x) + np.abs(y))))

from cs231n.data_utils import load_CIFAR10

def get_CIFAR10_data(num_training=49000, num_validation=1000, num_test=1000):
    """
    Load the CIFAR-10 dataset from disk and perform preprocessing to prepare
    it for the two-layer neural net classifier. These are the same steps as
    we used for the SVM, but condensed to a single function.  
    """
    # Load the raw CIFAR-10 data
    cifar10_dir = 'cs231n/datasets/cifar-10-batches-py'
    
    # Cleaning up variables to prevent loading data multiple times (which may cause memory issue)
    try:
       del X_train, y_train
       del X_test, y_test
       print('Clear previously loaded data.')
    except:
       pass

    X_train, y_train, X_test, y_test = load_CIFAR10(cifar10_dir)
        
    # Subsample the data
    mask = list(range(num_training, num_training + num_validation))
    X_val = X_train[mask]
    y_val = y_train[mask]
    mask = list(range(num_training))
    X_train = X_train[mask]
    y_train = y_train[mask]
    mask = list(range(num_test))
    X_test = X_test[mask]
    y_test = y_test[mask]

    # Normalize the data: subtract the mean image
    mean_image = np.mean(X_train, axis=0)
    X_train -= mean_image
    X_val -= mean_image
    X_test -= mean_image

    # Reshape data to rows
    X_train = X_train.reshape(num_training, -1)
    X_val = X_val.reshape(num_validation, -1)
    X_test = X_test.reshape(num_test, -1)

    return X_train, y_train, X_val, y_val, X_test, y_test

Tune your hyperparameters¶

What's wrong?. Looking at the visualizations above, we see that the loss is decreasing more or less linearly, which seems to suggest that the learning rate may be too low. Moreover, there is no gap between the training and validation accuracy, suggesting that the model we used has low capacity, and that we should increase its size. On the other hand, with a very large model we would expect to see more overfitting, which would manifest itself as a very large gap between the training and validation accuracy.

Explain your hyperparameter tuning process below.

$\color{blue}{\textit Your Answer:}$

The hyperparameter tuning process is exactly the same as done in Linear SVM and softmax. We choose an upper bound and lower bound for the learning rate and the regularization, and generate an equally spaced amount of hyperparameters to test over, and test every possible combination. Then we print them all out to observe which ones did well, and which ones did poorly. We then continue to refine the upper and lower bounds until we get successful training.

best_val = -1

results = {}
# We'll generate equally spaced hyperparameters and test over all of them.
# This defines the length of the lists that we'll test over.
num_param = 3
# Then we'll set the upper and lower bounds for each of our hyperparameters.
rate_low = 1e-3
rate_high = 5e-3
reg_low = 0.4
reg_high = 0.6
# Defining the new lists.
learning_rates = np.arange(rate_low, rate_high, (rate_high - rate_low)/num_param)
regularization_strengths = np.arange(reg_low, reg_high, (reg_high - reg_low)/(2*num_param))

for rate in learning_rates:
    for reg in regularization_strengths:
        # Generate the network.
        net = TwoLayerNet(input_size, hidden_size, num_classes)

        # Train the network
        stats = net.train(X_train, y_train, X_val, y_val,
                    num_iters=1000, batch_size=200,
                    learning_rate=rate, learning_rate_decay=0.95,
                    reg=reg)
        
        # After training the network, predict on the training data and validation data.
        y_pred_train = net.predict(X_train)
        y_pred_val = net.predict(X_val)
        # We can also score them on how well they did.
        y_train_score = np.mean(y_pred_train == y_train)
        y_val_score = np.mean(y_pred_val == y_val)
        # Storing the training and validation accuracy. 
        results[(rate, reg)] = (y_train_score, y_val_score)
        
        if y_val_score > best_val:
            best_val = y_val_score
            best_net = net
            best_statistics = stats

# Print out results.
for lr, reg in sorted(results):
    train_accuracy, val_accuracy = results[(lr, reg)]
    print('lr %e reg %e train accuracy: %f val accuracy: %f' % (
                lr, reg, train_accuracy, val_accuracy))

Inline Question

Now that you have trained a Neural Network classifier, you may find that your testing accuracy is much lower than the training accuracy. In what ways can we decrease this gap? Select all that apply.

  1. Train on a larger dataset.
  2. Add more hidden units.
  3. Increase the regularization strength.
  4. None of the above.

$\color{blue}{\textit Your Answer:}$ 3. Increase the regularization strength.

$\color{blue}{\textit Your Explanation:}$

If our testing accuracy is much lower than the training accuracy, it tells us we're probably overfitting the data. So, training on larger datasets doesn't really help us, since we're already pretty good at predicting on the training data. Adding more units increases the complexity of our network, and actually causes the overfitting problem to get worse if we maintain the same training set.
